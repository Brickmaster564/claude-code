# Channel-Specific Output Specs

Format constraints, character limits, and structure requirements per channel.

---

## Meta Ads (Facebook / Instagram)

### Format: Single Image / Carousel

| Field | Visible limit | Hard limit | Notes |
|-------|--------------|------------|-------|
| **Primary text** | ~125 chars (before "See more") | 1,000 chars | Front-load the hook in first 125 chars |
| **Headline** | ~40 chars | 255 chars | Keep under 40 for full visibility |
| **Description** | ~30 chars | 255 chars | Often hidden on mobile — don't rely on it |

### Structure

```
[HOOK — first line must stop the scroll, visible before "See more"]

[BODY — agitate, build desire, present solution]

[CTA LINE — clear action with risk reversal]

HEADLINE: [short, punchy, benefit-driven]
DESCRIPTION: [supporting detail or urgency]
```

### Rules
- Hook must work standalone (most people don't click "See more")
- One idea per ad. Don't cram multiple messages.
- Use line breaks for readability
- Emojis: sparingly, only if appropriate for the brand
- No ALL CAPS unless one word for emphasis

## Google Responsive Search Ads (RSAs)

### Format

| Field | Limit | Count |
|-------|-------|-------|
| **Headlines** | 30 chars each | Up to 15 (min 3) |
| **Descriptions** | 90 chars each | Up to 4 (min 2) |

### Rules
- Each headline must make sense on its own and in any combination with other headlines
- Include the primary keyword in at least 2 headlines
- Mix headline types:
  - Benefit-driven ("Save Up to 40% on Life Insurance")
  - Feature-driven ("Free Quote in 60 Seconds")
  - CTA-driven ("Get Your Free Quote Today")
  - Trust-driven ("Rated 4.9 Stars by 2,000+ Families")
  - Urgency-driven ("Limited Time — Lock In Your Rate")
- Descriptions expand on headlines — add proof, mechanism, risk reversal
- Pin critical headlines to positions 1-2 only if necessary

### Output format
```
HEADLINES:
H1: [30 chars max]
H2: [30 chars max]
...

DESCRIPTIONS:
D1: [90 chars max]
D2: [90 chars max]
...
```

## Landing Page

### Structure (top to bottom)

| Section | Purpose | Key elements |
|---------|---------|-------------|
| **Hero** | Stop, hook, promise | Headline, subheadline, CTA, hero image direction |
| **Problem** | Agitate the pain | 3-5 pain points using VOC language |
| **Solution** | Present the offer | What it is, how it works, what they get |
| **Mechanism** | Build belief in the approach | Why this works when other things haven't |
| **Proof** | Eliminate doubt | Testimonials, stats, case studies, logos |
| **Offer stack** | Make the value obvious | Everything included, bonuses, value anchoring |
| **Risk reversal** | Remove last objection | Guarantee, free trial, no obligation |
| **CTA** | Drive action | Button text, supporting microcopy |
| **Urgency** | Push the fence-sitters | Deadline, scarcity, or consequence of inaction |
| **FAQ** | Handle remaining objections | Top 5-7 objections reframed as questions |

### Rules
- Hero headline: max 10 words. Must communicate the core benefit.
- Subheadline: supports the headline, adds specificity or mechanism
- CTA button text: action verb + what they get ("Get My Free Quote")
- Repeat CTA at least 3 times throughout the page
- Every section should be skimmable — headers, bullets, short paragraphs

## Email Sequence

### Format per email

| Field | Limit | Notes |
|-------|-------|-------|
| **Subject line** | ~50 chars | Must create curiosity or urgency |
| **Preview text** | ~90 chars | Complements subject, shown in inbox |
| **Body** | Varies | Short paragraphs, conversational, one CTA per email |

### Sequence structure (lead nurture)

| Email | Timing | Purpose | Angle |
|-------|--------|---------|-------|
| 1 | Immediately | Deliver lead magnet + set expectations | Value + mechanism |
| 2 | Day 1 | Story + pain agitation | Pain / fear |
| 3 | Day 2 | Social proof + results | Proof |
| 4 | Day 3 | Overcome #1 objection | Objection handling |
| 5 | Day 4 | Urgency / final push | Urgency + aspiration |

### Rules
- Subject lines: no clickbait that doesn't pay off in the body
- One CTA per email, repeated 2-3 times
- Write like a human, not a brand. First person, conversational.
- P.S. lines are high-read — use for secondary hook or CTA restatement

## VSL (Video Sales Letter)

### Structure

| Section | Duration | Purpose |
|---------|----------|---------|
| **Hook** | 0-5 sec | Pattern interrupt — must stop the scroll |
| **Open loop** | 5-15 sec | Create information gap they need to close |
| **Problem** | 15-60 sec | Agitate with VOC language |
| **Story/mechanism** | 1-3 min | Why this solution is different |
| **Proof** | 30-60 sec | Testimonials, stats, results |
| **Offer** | 30-60 sec | What they get, stack the value |
| **Close** | 15-30 sec | CTA + urgency + risk reversal |

### Rules
- Hook must work with sound OFF (text overlay / visual)
- Write for spoken language — read it out loud
- One sentence per line in the script
- Mark visual cues: [TEXT ON SCREEN: "..."], [CUT TO: testimonial]
- Open loop must create a question the viewer needs answered

### Output format
```
[HOOK — 0:00-0:05]
[Text on screen: "..."]
[Spoken: "..."]

[OPEN LOOP — 0:05-0:15]
[Spoken: "..."]

...
```

## Social Media Captions (Nalu)

### Format

Repurposed from podcast/interview/video transcripts. One caption per post.

### Rules
- Open with a hook line that stops the scroll. No throat-clearing.
- Buttery, conversational, 6th-grade reading level.
- Short paragraphs, 1-2 sentences max per block.
- Line breaks between every thought for mobile readability.
- End with a CTA or conversation starter (question, prompt to comment/share/save).
- No hashtag stuffing. Max 3-5 relevant hashtags if needed, placed at the end or in first comment.
- Match the tone of the source content, not a generic brand voice.
- Pull direct quotes or paraphrased gold from the transcript.

### Output format
```
[HOOK LINE]

[Body — 3-6 short paragraphs pulling from transcript]

[CTA / conversation starter]

[Hashtags if applicable]
```

## Newsletters (Nalu)

### Format

Repurposed from podcast/interview/video transcripts. Email newsletter format.

### Rules
- Subject line: max 50 chars, curiosity or value-driven. No clickbait that doesn't pay off.
- Preview text: max 90 chars, complements the subject line.
- Open with a hook, not a greeting. "Hey [name]" is fine but the next line must earn the read.
- Conversational, first-person tone. Reads like a smart friend sharing an insight.
- Pull the best insight, story, or takeaway from the transcript and build the newsletter around it.
- One core idea per newsletter. Don't try to cover everything from the episode.
- Short paragraphs. Single sentences get their own line.
- End with a clear CTA: listen to the episode, reply, share, click a link.
- P.S. line for a secondary hook or CTA.

### Output format
```
SUBJECT: [50 chars max]
PREVIEW: [90 chars max]

[Opening hook]

[Body — built around one core insight from the transcript]

[CTA]

P.S. [Secondary hook or CTA]
```

## Short-Form Video Scripts (Nalu)

### Format

Repurposed from podcast/interview/video transcripts. For Reels, TikTok, YouTube Shorts. 15-60 seconds.

### Rules
- Hook in the first 1-2 seconds. Must work visually (text on screen) and spoken.
- One idea per script. If there are multiple good moments, make multiple scripts.
- Write for spoken delivery, not reading. Short sentences. Natural cadence.
- Open loops early: tease what's coming so they don't swipe away.
- End with a payoff that delivers on the hook's promise.
- Include text-on-screen directions where they add impact.
- Keep total word count under 150 for 60-second scripts, under 40 for 15-second.

### Output format
```
[HOOK — 0:00-0:02]
[Text on screen: "..."]
[Spoken: "..."]

[BODY — 0:02-0:50]
[Spoken: "..."]

[PAYOFF — 0:50-0:60]
[Spoken: "..."]
[Text on screen: "..." (if applicable)]
```

## Announcement Copy (Nalu)

### Format

New episode drops, guest announcements, milestone posts, event promotions. Can be used across social, email, or both.

### Rules
- Lead with the hook, not the announcement. "We just dropped a new episode" is boring. Lead with the most compelling thing from it.
- Make the reader want to consume the content, not just know it exists.
- Pull the single most quotable or surprising line from the transcript if available.
- Keep it punchy. Announcement copy should be shorter than regular captions.
- Clear CTA: link to listen, watch, sign up, etc.

### Output format
```
[HOOK — the most compelling thing]

[1-2 lines of context / what they'll get from it]

[CTA with link placeholder]
```

## Channel Selection Guide

| If the ask is... | Default to... |
|-----------------|---------------|
| "Write me some ads" | Meta ads (ask to confirm) |
| "Headlines for a campaign" | Meta ad headlines OR Google RSAs (ask which) |
| "Landing page copy" | Landing page full structure |
| "Email sequence" | 5-email nurture sequence |
| "Video script" | VSL format (Client Network) or Short-form script (Nalu) |
| "Sales letter" | Landing page (long-form) |
| "Caption" / "social post" | Social media caption (Nalu) |
| "Newsletter" | Newsletter (Nalu) |
| "Script" / "reel" / "short" | Short-form video script (Nalu) |
| "Announcement" / "episode drop" | Announcement copy (Nalu) |

If the channel is unclear, ask. Don't assume.
